creator_ai · 灵境引擎

让 AI 直接操控 Cocos Creator

creator_ai 是基于 MCP（Model Context Protocol）协议的 Cocos Creator 编辑器插件。 它将引擎 API 封装为 16 个标准化工具，让 Cursor、Claude 等 AI Agent 能够读取场景、编辑节点、 管理脚本、导入 DSL Bundle — 真正实现 AI 驱动的游戏开发。

GitHub 仓库 技术文档
🚀

零配置启动

安装插件后，通过 Creator 菜单一键启动 MCP 服务，AI Agent 即可连接。

🎮

全场景覆盖

支持场景编辑、Prefab 编辑、资源管理、脚本开发等完整工作流。

📐

DSL 驱动

JSON DSL 描述 UI，AI 生成 Bundle 后一键导入为 Cocos Prefab。

🛡️

安全可控

白名单字段修改、dryRun 预览、操作日志，确保 AI 操作可审计。

快速开始

三步接入 MCP

从安装到 AI 操控引擎，只需几分钟

01

安装插件

client/extensions/creator_ai/
02

启动 MCP 服务

AI Tools → MCP 服务 → 启动
03

连接 AI Agent

http://localhost:3105/sse
MCP 工具

16 个标准化工具

覆盖场景编辑、资源管理、脚本开发、诊断修复全流程

基础 · 2 场景 · 6 资源 · 3 开发 · 3 诊断 · 2
ping 基础

健康检查，验证 MCP 连接是否正常。

Phase 1
get_project_info 基础

获取当前 Cocos Creator 项目信息（名称、路径、引擎版本）。

Phase 1
get_editor_context 场景

获取编辑器场景上下文，包括场景名、根节点、脏标记等。

Phase 1
query_scene_hierarchy 场景

查询场景或 Prefab 编辑模式下的节点树结构（深度上限 5）。

Phase 1
query_node_state 场景

按 UUID 查询节点 Transform / UITransform / Sprite 摘要。

Phase 1
apply_node_patch 场景

对白名单字段批量修改节点属性，支持 dryRun 预览。

Phase 1
save_editor_scene 场景

保存当前场景或正在编辑的 Prefab。

Phase 1
toggle_project_preview 场景

启动或刷新 Cocos Creator 游戏预览。

Phase 1
list_assets 资源

列出 db:// 目录下的资源列表，每次最多 200 条。

Phase 1
import_dsl_bundle 资源

将 DSL Bundle 目录导入项目，自动生成 Prefab 并合并资源。

Phase 1
get_last_dsl_log 资源

获取最近一次 DSL 导入日志（状态、路径、错误信息）。

Phase 1
script_manager 开发

创建、读取、更新、删除、列出 TypeScript 脚本，支持自动编译。

Phase 2
component_manager 开发

管理节点组件：挂载/卸载、读写属性、创建/删除节点。

Phase 2
engine_control 开发

引擎级操作：状态查询、脚本刷新、缓存清理、编译状态。

Phase 2
console_reader 诊断

读取编辑器控制台日志，支持按级别和关键词过滤。

Phase 3
scene_file_editor 诊断

直接读写 .scene JSON 文件，支持批量修改场景数据。
